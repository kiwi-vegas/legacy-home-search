# Legacy Home Search — Content Pipeline

## Overview

This document is the single source of truth for the full content operation — from automated research through idea scoring, editorial review, thumbnail creation, website publishing, and Facebook distribution.

**Stack:** Next.js 16 · Sanity CMS · Upstash Redis · Claude (Anthropic) · Blotato (Facebook)  
**Market:** Virginia Beach, Chesapeake, Norfolk, Suffolk, Hampton, Newport News (Hampton Roads, VA)  
**Sanity project ID:** `2nr7n3lm`  
**Production URL:** `https://legacyhometeamlpt.com`

## Full End-to-End Flow

```
Daily / Weekly Research (Cron)
  └─ Tavily fetches 8–10 article searches (3 pinned + 5–7 rotating)
  └─ Renick Blog analytics scrape (weekly) — pattern extraction
  └─ All scored → stored as ideas in Redis (lhs:ideas:queue)

Weekly Idea Digest (Cron — Thu/Fri 8 AM PT)
  └─ Top ideas emailed to kiwi@ylopo.com
  └─ /admin/idea-review?secret=… — approve / skip / defer ideas

Operator approves an idea on /admin/idea-review
  └─ /api/content/ideas/approve (POST)
       └─ Claude Sonnet writes full blog post from idea
       └─ Sanity doc created: workflowStatus: 'media_pending' (hidden from public)

VA opens /admin/va-queue
  └─ Sees all media_pending and media_ready posts
  └─ Opens post → /admin/va-queue/[postId]
       └─ Generates AI thumbnail OR uploads image
       └─ Reviews/edits Facebook caption (auto-generated by Claude Haiku)
       └─ Clicks "Mark Ready" → workflowStatus: 'media_ready'
       └─ Clicks "Publish"
            └─ Sanity: coverImage saved, workflowStatus: 'published' (website live)
            └─ Blotato: POST /v1/posts → Facebook page
            └─ postSubmissionId stored on Sanity doc
            └─ Poll Blotato every 10s → blotatoPublishStatus + facebookPostUrl stored

Social Queue (VA queue bottom section)
  └─ Shows posts published in the last 21 days with no facebookPostUrl
  └─ "Post to Facebook" → auto-generates caption → VA reviews → publishes social only
  └─ "Decline" → marks socialDeclined: true, hides from queue permanently
```

## Required Evergreen Topics (Coverage System)

A registry of high-performing post templates that **must be covered** for every Hampton Roads city the client serves. The system audits Sanity, identifies gaps, and seeds them as IdeaCandidates so they appear on `/admin/idea-review` for approval — without ever duplicating a post that already exists.

### Required topic templates

| Template | Cities | Audience | Category |
|----------|--------|----------|----------|
| `What Does It Cost to Buy a Home in [City] in 2026?` | Hampton Roads + 6 cities | Buyer | cost-breakdown |
| `What Does It Cost to Sell a Home in [City]?` | Hampton Roads + 6 cities | Seller | cost-breakdown |
| `How Do Virginia Property Taxes Work for [City] Home Buyers?` | Hampton Roads + 6 cities | Buyer | buying-tips |
| `[City A] vs [City B]: Which Is Better for [Buyers/Investors] in 2026?` | 5 priority pairs × 2 audiences | Buyer/Investor | community-spotlight |
| `Is 2026 a Good Time to Buy in [City]?` | Hampton Roads + 6 cities | Buyer | buying-tips |
| `What Happens After Your Offer Is Accepted in Virginia?` | State-level (1 post) | Buyer | buying-tips |
| `What Do Flood Zones Mean for Home Buyers in [City]?` | Coastal cities (5 cities) | Buyer | flood-and-risk |

**44 total topic goals** across all templates and city/audience combinations.

### How coverage detection works

`lib/required-topics.ts` defines each goal with `matchAllOf: string[]` — a list of substrings that must ALL appear in a published post's title or slug for the goal to be considered covered. Disambiguation is built in (e.g., a "Hampton Roads" post does not falsely match a "Hampton" city goal).

### How gap seeding works

`/api/cron/required-topics-coverage` runs on the **3rd of each month at 9 AM PT** (also POST-triggerable manually). For each goal:
- ✅ **Covered by published post** → skip
- ⏳ **Already pending in idea queue** → skip
- 🆕 **Missing AND not queued** → seed as `IdeaCandidate` with `source: 'internal'`, score ~70, marked `Required Evergreen`

The seeded ideas land on `/admin/idea-review` like any other idea. Approving one writes the post (using the `brief` and `targetKeyword` from the goal) and routes it through the standard VA queue → publish flow.

### Manual operations

```bash
# Dry run — preview gaps without seeding
curl -X POST "https://www.legacyhometeamlpt.com/api/cron/required-topics-coverage?secret=ADMIN_SECRET&dryRun=1"

# Run — seed any missing goals into the idea queue
curl -X POST "https://www.legacyhometeamlpt.com/api/cron/required-topics-coverage?secret=ADMIN_SECRET"
```

### Adding new required topic templates

Edit `lib/required-topics.ts` → append a generator to `REQUIRED_TOPICS`. The next cron run picks up the new goals and seeds gaps. Existing posts and queued ideas are unaffected.

### Files

| File | Purpose |
|------|---------|
| `lib/required-topics.ts` | Registry: defines all topic templates and city expansions |
| `lib/required-topics-coverage.ts` | `checkCoverage()` + `seedMissingTopics()` |
| `app/api/cron/required-topics-coverage/route.ts` | GET (cron) and POST (manual) endpoint |

## Phase 1: Research & Idea Engine

### Research Cron (`/api/cron/research`)

Runs daily at 6 AM PT. Searches Tavily for Hampton Roads real estate news, scores results against the 100-point model, and stores passing ideas in Redis.

**Pinned daily queries (always run):**
```
'Virginia Beach real estate market news 2026'
'Hampton Roads housing market trends 2026'
'Hampton Roads community development news projects 2026'
```

**Rotating query pool (42 topics, 5 slots/day cycle through):**
- Virginia Beach home prices / Chesapeake / Norfolk / Suffolk / Newport News / Hampton real estate
- Hampton Roads rental market / investment returns / short-term rental regulations / Airbnb
- Virginia HOA law / property tax / zoning / landlord-tenant law
- Hampton Roads major development / Port of Virginia / resort area projects / downtown arena
- Military: PCS season / base expansion / VA loans / BAH rates / NAS Oceana / Norfolk Naval Station
- Virginia Beach oceanfront / waterfront / new construction neighborhoods
- First-time buyer programs / VHDA grants / down payment assistance
- Flood insurance / FEMA flood zone / sea level rise / elevation certificates
- Homeowner insurance rates / wind and storm coverage in coastal Virginia
- Cost of ownership breakdown / closing costs / property tax comparison across Hampton Roads cities
- Hampton Roads local events / festivals / economic activity signals
- New business openings / corporate relocations signaling housing demand
- 55+ retirement communities / senior housing in Hampton Roads
- Schools and school district ratings for family buyers
- Rent vs. buy analysis for Hampton Roads market

### Renick Pipeline (`/api/cron/renick-pipeline`)

Scrapes the Renick Blog Analytics dashboard via Tavily HTML extraction (no API). Extracts top-performing post patterns (title formats, categories, CTR lift). Converts patterns into idea candidates scored by the 100-point model. Feeds into the same Redis idea queue — no separate pipeline.

### Idea Scoring Model (100 points)

| Dimension | Max | Description |
|-----------|-----|-------------|
| Local Relevance | 25 | Hampton Roads / Virginia-specific, not national filler |
| Timeliness | 20 | Time-sensitive or evergreen with fresh angle |
| Format Fit | 15 | Matches proven formats (cost breakdown, Q&A, how-to, comparison) |
| Audience Value | 15 | Directly useful to Hampton Roads buyers and homeowners |
| Source Credibility | 10 | Government data, local publication, established real estate source |
| Novelty | 10 | Not a repeat of recent content; new angle |
| SEO Potential | 5 | Matches how people actually search |

**Pass threshold:** ≥55/100 stored in Redis. Ideas scoring ≥85 with `timeliness: 'breaking'` flag trigger an immediate urgent email to kiwi@ylopo.com, bypassing the weekly queue.

### Redis Idea Store

Key: `lhs:ideas:queue` (sorted set, score = idea score desc)  
TTL: 7 days  
Each entry: `{ id, title, url, excerpt, category, score, breakdown, source, timeliness, savedAt }`

## Phase 2: Idea Review

### `/admin/idea-review`

The operator reviews scored ideas, approves ones worth writing, and skips/defers others. This is the only gate before a post gets written — Claude writes nothing until an idea is approved here.

**Actions per idea:**
- **Approve** → triggers post writing immediately (see Phase 3)
- **Skip** → removes from queue permanently
- **Defer** → keeps in queue for next week's digest

**"Generate More Ideas" button:** Manually triggers `/api/content/ideas/generate` — runs `fetchAndScoreIdeas()` on demand and stores results without waiting for the daily cron.

### Idea Digest Email (Weekly)

Sent Thursday or Friday at 8 AM PT to `kiwi@ylopo.com`. Lists top 10–15 scored ideas with title, score, category, source, and a direct link to `/admin/idea-review`. The email replaces the old daily digest and the old Renick approval email.

## Phase 3: Post Writing

### `/api/content/ideas/approve`

Called when the operator clicks Approve on the idea-review page. Claude Sonnet 4.6 writes a full blog post from the idea's title, excerpt, and URL. Post is saved to Sanity CMS with `workflowStatus: 'media_pending'` — hidden from the public blog until the VA adds a thumbnail and publishes.

**Post writing rules:**
- Voice: Virginia Beach real estate professional at Legacy Home Search — knowledgeable, approachable, never salesy
- Ties national/Virginia news back to the local Hampton Roads market
- Structure: engaging headline → intro → 2–3 body sections (`##` headings) → `## What This Means For You` (3–4 bullets) → closing → source credit (linked)
- All posts must answer: "What does this mean for Hampton Roads buyers and homeowners?"

## Phase 4: VA Media Queue

### `/admin/va-queue`

The VA's home base. Shows all posts waiting to be finished and published.

**Two sections:**
- **Needs Media** — `workflowStatus: 'media_pending'` posts (amber badge, no thumbnail yet)
- **Ready to Publish** — `workflowStatus: 'media_ready'` posts (thumbnail saved, one click to publish)
- **Published — Post to Facebook** — `workflowStatus: 'published'` posts from the last 21 days with no `facebookPostUrl` and `socialDeclined != true` (social queue — see Phase 6)

### `/admin/va-queue/[postId]`

The media editor. Three panels:

**Panel 1 — Article Context (read-only)**
Title, category, excerpt. Auto-generated Facebook caption (editable).

**Panel 2 — Thumbnail Builder**
- Prompt editor: preloaded with article context merged into a template
- Background picker: community photo grid (`public/community-photos/[city]/`)
- Click **✨ Generate Thumbnail** → calls `/api/blog/generate-thumbnail`
- Or click **Upload Image** for a manual override

**Panel 3 — Publish Actions**
- **✓ Mark as Ready** → `workflowStatus: 'media_ready'`
- **🚀 Publish** → publishes to website + Facebook simultaneously (see Phase 5)
- Inline status: spinning → Published ✓ with links, or Failed ✗ with retry

## Phase 5: Publishing

### What happens when the VA clicks Publish

1. `workflowStatus` set to `'publishing'`
2. Social copy finalized (VA-edited or auto-generated by Claude Haiku)
3. Sanity image URL resolved at 1200px width
4. Blotato POST `/v1/posts` called with `accountId`, `pageId`, `text + url`, `imageUrl`
5. `postSubmissionId` stored on Sanity doc
6. `workflowStatus` set to `'published'` — post is now live on website
7. Client polls `/api/content/blotato-status` every 10s (up to 30 attempts / ~5 minutes)
8. On resolution: `blotatoPublishStatus`, `blotatoPublishedAt`, `facebookPostUrl` stored on doc

### Blotato API

```
POST /v1/posts
Authorization: Bearer {BLOTATO_API}
{
  accountId: "27245",
  pageId: "1101893253009079",
  content: {
    text: "<social copy>\n\nhttps://legacyhometeamlpt.com/blog/<slug>",
    imageUrl: "<Sanity CDN image URL>"
  }
}
→ { postSubmissionId: "abc123" }

GET /v1/posts/abc123
→ { status: "published" | "failed", postUrl?: "..." }
```

### Workflow Status Model

| Status | Visible on website? | Meaning |
|--------|--------------------|----|
| `media_pending` | No | Post written; VA needs to add thumbnail |
| `media_ready` | No | Thumbnail saved; ready to publish |
| `publish_pending` | No | Publish triggered, not yet confirmed |
| `publishing` | No | Blotato call in progress |
| `published` | Yes | Live on website + Facebook |
| `publish_failed` | No | Blotato failed; VA can retry |

**Legacy compatibility:** Old posts with `status: 'published'` remain live. GROQ filter: `workflowStatus == 'published' || status == 'published'`.

## Phase 6: Social Queue (Backfill Facebook for Existing Posts)

The VA Queue's bottom section surfaces posts that are already live on the blog but haven't been posted to Facebook.

**Query:** `workflowStatus == 'published'` AND `publishedAt > [21 days ago]` AND `!defined(facebookPostUrl)` AND `socialDeclined != true`

**Per-post actions:**
- **Post to Facebook** → generates caption via Claude Haiku → VA reviews in inline editor → submits → calls `publishSocialOnly()` — this does NOT re-publish the website post or change `workflowStatus`
- **Decline** → sets `socialDeclined: true` on Sanity doc → post disappears from queue permanently

`publishSocialOnly()` only patches: `blotatoPostSubmissionId`, `blotatoPublishStatus`, `blotatoPublishedAt`, `facebookPostUrl`, `socialCopy`. The `workflowStatus` is never touched.

## Image Generation Pipeline

### Thumbnail generation (AI)

Primary: **OpenAI gpt-image-1** (`images.edit()`) — Barry's photo (`public/barry-transparent.png`) composited over a community background. GPT-4o writes the design prompt from article context.

Fallback chain:
1. gpt-image-1 (primary)
2. Gemini — community background scenes
3. OG image scraped from article URL
4. Unsplash API (requires `UNSPLASH_ACCESS_KEY`)
5. Fallback image pool — pre-curated Unsplash URLs, deterministic pick by article URL hash

### Community background photos

Static manifest in `lib/thumbnail-asset-resolver.ts` — all filenames hardcoded per community slug. No `fs.readdirSync` calls. This is required for Vercel serverless compatibility (prevents 177MB of community photos from being bundled into the function).

When adding new community photos to `public/community-photos/[city]/`, update the manifest in `lib/thumbnail-asset-resolver.ts`.

### Market report covers

**DALL-E 3** via `lib/image-gen-market-report.ts` — YouTube/financial illustration style. Separate from blog thumbnails.

## Blog Post Page (`/blog/[slug]`)

**Hero image:** `objectFit: 'cover'`, `objectPosition: 'top'`, `max-height: 48vh`. Anchors the top of the image so text/graphic elements in the upper portion stay visible.

**Community listings widget:** `BlogCommunityListings` renders below the post body. `detectCommunities(title, slug)` matches post content to Hampton Roads cities and shows the relevant YLOPO listings widget (or a Hampton Roads-wide widget if no specific city is detected).

### External Link Policy

Every blog post writer (real estate posts in `lib/writer.ts`, events posts in `lib/events-research.ts`) automatically links named entities to their official websites. All links open in a new tab via `target="_blank"` in `components/PortableText.tsx`.

**What gets linked (first mention only):**
- Named events and festivals (e.g., Chesapeake Jubilee, Town Point Wine Festival)
- Named venues and parks (e.g., Town Point Park, Chesapeake City Park)
- Government agencies and programs (e.g., VHDA, City of Norfolk, Virginia Beach City Public Schools)
- Military installations (e.g., Naval Station Norfolk, NAS Oceana)
- Named businesses and organizations with public websites

**Rules baked into the Claude prompts:**
- Only link to URLs Claude is confident are correct and official — never invented or guessed
- Link only the FIRST mention of each entity, not every occurrence
- Skip the link if unsure of the official URL

**Portable Text link format (Sanity):**
Claude writes links as markdown `[Entity Name](https://url.com)` in the body string. `lib/portable-text-utils.ts` → `markdownToPortableText()` converts these to Sanity `markDefs` with `_type: 'link'`. The renderer picks them up automatically.

**Retroactive enrichment:**
```bash
npx tsx --env-file=.env.local scripts/add-links-to-posts.ts
```
Updates the last 6 posts (all statuses). Safe to re-run — Claude is instructed to preserve all existing links during enrichment.

## Market & Research Priorities

**ALL articles must be about Virginia Beach, Chesapeake, Norfolk, Suffolk, Hampton Roads, or directly relevant Virginia law/policy.** Articles about other states score ≤1 and are dropped.

High-value topic weights in the scoring model:

1. **Hampton Roads property values & investment** — home price forecasts, rental returns, neighborhood comparisons
2. **Virginia law changes** — property tax, HOA regulations, zoning, tenant/landlord, real estate transactions
3. **Major development projects** — Port of Virginia, resort area, downtown arena, corporate relocations
4. **Military sector** — base expansion, PCS season, VA loans, BAH rates, NAS Oceana
5. **Flood & coastal risk** — FEMA flood zones, flood insurance, sea level rise (Norfolk is one of the fastest-sinking US cities)
6. **Cost breakdowns** — closing costs, HOA fees, property taxes, insurance — Renick's #1 format by engagement lift

## Enhanced Category Strategy

*Based on Renick Blog Effectiveness Dashboard analysis — April 2026*

**`cost-breakdown`** — Every post includes a summary table in the first third. Renick's #1 format (5 of his top 10 posts).

**`flood-and-risk`** — Flood zone classification, insurance costs, sea level rise. Renick's single highest-lift post was an insurance explainer (+1829%). Currently underrepresented.

### Format Rules (All Categories)

1. **Question-led titles by default.** "What Does It Cost…", "How Do… Work?", "Is X or Y Better?" Only use statement titles when question form is awkward.
2. **Include a real number in the title** whenever possible.
3. **Cost breakdown posts:** Summary table required, placed in the first third.
4. **Comparison posts:** Must end with an explicit "Best For…" conclusion.
5. **Process guides:** Numbered steps, not prose. Each step: what happens, who does it, how long.
6. **All market posts:** "What This Means For Hampton Roads Buyers/Sellers" is non-negotiable.
7. **Military-angle posts:** Specify active duty vs. veterans vs. both. BAH rates, VA loan entitlement, and PCS timelines vary.

socialCopy: string                 // Facebook caption text

blotatoPublishStatus: string       // 'pending' | 'published' | 'failed'
blotatoPublishedAt: datetime
facebookPostUrl: string            // Blotato resolves this on success

## Adapting This Pipeline for a New Client

1. **New Sanity project** — create project, update `NEXT_PUBLIC_SANITY_PROJECT_ID` and `NEXT_PUBLIC_SANITY_DATASET`
2. **Market geography** — update pinned queries and rotating pool in `lib/research.ts` to match the client's metro area
3. **Agent voice** — update the system prompt in `lib/writer.ts` with the agent's name, years of experience, local anchors, and specialty
4. **Social copy persona** — update the Claude Haiku prompt in `lib/publish-service.ts` → `generateSocialCopy()`
5. **Blotato credentials** — new `BLOTATO_API`, `BLOTATO_ACCOUNT_ID`, `BLOTATO_FACEBOOK_PAGE_ID` for the client's Facebook Page
6. **Community photos** — add `public/community-photos/[city]/` assets for the new market + update the static manifest in `lib/thumbnail-asset-resolver.ts`
7. **Agent photo** — replace `public/barry-transparent.png` with the new agent's transparent PNG
8. **Production URL** — update `NEXT_PUBLIC_APP_URL` in Vercel env vars
9. **Emails** — update `OPERATOR_EMAIL` and `BARRY_EMAIL` in Vercel env vars
10. **Blog post categories** — verify the category list in `sanity/schema/blogPost.ts` matches the client's content strategy; add/remove as needed
11. **YLOPO widget** — update `detectCommunities()` in `app/(site)/blog/[slug]/page.tsx` to match the new market's city names

**Phase 2 multi-client (future):** A Sanity `clientConfig` document will store Blotato credentials, agent photo, Facebook page ID, and market config per client. `lib/blotato-client.ts` is designed so `getAccountId()` / `getPageId()` can be swapped from env vars to Sanity config without changing the publish call.
